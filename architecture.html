System Overview

A modern, full-stack web application for factory inventory management, order tracking, and financial analytics. Built with Vue 3 on the frontend and Python FastAPI on the backend, featuring in-memory data persistence and real-time filtering capabilities.

Key Characteristics

  • Client-Server Architecture: Separation of concerns with dedicated frontend and backend
  • Stateless API: RESTful endpoints with query parameter filtering
  • In-Memory Data: JSON-based mock data loaded at startup (no database)
  • Real-Time Filtering: 4 global filters applied across views
  • Multi-Language Support: English and Japanese localization
  • Interactive Dashboard: Custom SVG charts and responsive layouts

Data Flow

Request-Response Cycle

  1. User Interaction: User selects filters (warehouse, category, status, month) in Vue component
  2. State Update: Vue composables update filter refs and emit URL query parameters
  3. API Request: Frontend calls api.js method (e.g., getOrders(filters))
  4. HTTP Request: Axios sends GET request to FastAPI endpoint with query params
  5. Backend Processing: FastAPI route handler receives request and applies filters
  6. Data Filtering: In-memory data filtered by warehouse, category, status, month (if applicable)
  7. Validation: Pydantic models validate response data structure
  8. JSON Response: Backend returns filtered data as JSON with 200 status
  9. Frontend Update: Vue updates refs with response data
  10. Computed Properties: Derived data (totals, percentages) recalculates automatically
  11. Template Re-render: Vue re-renders DOM with updated data

Filter System

  • 4 Global Filters: Time Period, Warehouse, Category, Order Status
  • Query Parameters: ?warehouse=A&category=Circuit&status=Delivered&month=2025-01
  • Selective Application: Each endpoint applies only relevant filters
  • Filter Logic: Backend checks for 'all' value to skip filtering
  • Case-Insensitive: Category filters use lowercase comparison
  • Time Support: Months (2025-01) and quarters (Q1-2025)

Application Features

Dashboard

Executive overview with key performance indicators, revenue metrics, inventory status, and order summary with visual charts

Inventory Management

Complete stock tracking across multiple warehouses with reorder points, unit costs, and real-time filtering by warehouse and category

Order Tracking

Monitor orders by status (Delivered, Shipped, Processing, Backordered) with dates, customer info, and filtering capabilities

Development & Deployment

Local Development

Backend (Python):
$ cd server
$ python -m venv venv
$ source venv/bin/activate # Windows: venv\Scripts\activate
$ pip install fastapi uvicorn pydantic
$ python main.py

Frontend (Node):
$ cd client
$ npm install
$ npm run dev

Ports:
Frontend: http://localhost:3000
Backend API: http://localhost:8001
API Docs: http://localhost:8001/docs

Production Considerations

  • Replace JSON mock data with persistent database (PostgreSQL, MongoDB)
  • Implement authentication and authorization
  • Add rate limiting and request validation
  • Restrict CORS to specific origins
  • Deploy frontend to CDN or static hosting
  • Deploy backend to containerized environment (Docker, Kubernetes)
  • Add comprehensive logging and monitoring
  • Implement caching strategy for performance
